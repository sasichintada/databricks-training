# 📘 SQL Practice Questions

## 📖 Overview

The exercises are based on an **Employees** dataset and cover essential SQL operations frequently used in data analysis, database management, and technical interviews.

The practice set focuses on retrieving, filtering, grouping, aggregating, and analyzing employee data using various SQL clauses and operators. By working through these queries, learners gain hands-on experience in writing efficient SQL statements and understanding how data can be manipulated and analyzed within relational databases.

---

## 🚀 Tasks Completed

### 1. SELECT Statements

Implemented queries to:

* Display all employee records.
* Retrieve specific columns such as employee names and salaries.
* Display employee names along with their departments.
* Filter and display employees from a particular department.
* Retrieve employee names and years of experience.

### 2. WHERE Clause

Implemented filtering operations to:

* Find employees earning above a specific salary.
* Retrieve employees based on city location.
* Filter employees according to work experience.
* Display employees belonging to a particular department.
* Search records matching exact salary values.

### 3. GROUP BY Clause

Performed aggregation operations including:

* Calculating total salary department-wise.
* Finding average salary per department.
* Counting employees in each city.
* Identifying maximum salary within departments.
* Finding minimum experience department-wise.

### 4. HAVING Clause

Applied conditions on grouped data to:

* Find departments with more than a specified number of employees.
* Identify departments with high average salaries.
* Determine cities with multiple employees.
* Find departments with large salary expenditures.
* Retrieve departments where the highest salary exceeds a threshold.

### 5. TOP / LIMIT Queries

Implemented ranking and limiting operations to:

* Display the highest-paid employees.
* Retrieve employees with the greatest experience.
* Find top salaries within specific departments.
* Limit records returned from a particular city.
* Identify the employee with the highest salary.

### 6. DISTINCT Keyword

Used DISTINCT to:

* Retrieve unique department names.
* Display unique city names.
* Find unique salary values.
* Display unique department-city combinations.
* Retrieve distinct experience levels.

### 7. Comparison Operators

Applied comparison operators such as:

* Greater than (`>`)
* Less than (`<`)
* Greater than or equal to (`>=`)
* Less than or equal to (`<=`)
* Not equal to (`<>`)

### 8. Logical Operators

Implemented logical conditions using:

* AND
* OR
* NOT

to combine multiple filtering criteria.

### 9. IN and NOT IN Operators

Used set-based filtering to:

* Retrieve records matching multiple values.
* Exclude records belonging to specific categories.

### 10. BETWEEN Operator

Performed range-based filtering on:

* Salary values
* Experience levels
* Employee IDs

using both BETWEEN and NOT BETWEEN conditions.

### 11. LIKE Operator

Implemented pattern matching to:

* Find names starting with specific characters.
* Find names ending with specific characters.
* Search names containing certain letters.
* Filter cities based on prefixes.
* Match departments based on suffix patterns.
